                  3 %
                  4 % For each binSize x binSize region in an image I, computes a histogram of

                 13 %   N. Dalal and B. Triggs, "Histograms of Oriented
                 14 %   Gradients for Human Detection," CVPR 2005.
                 15 % Setting parameters appropriately gives almost identical features to the
                 16 % original HOG or updated FHOG features, see hog.m and fhog.m for details.
                 17 %
                 18 % The input to the function are the gradient magnitude M and orientation O
                 19 % at each image location. See gradientMag.m for computing M and O from I.
                 20 %
                 21 % The first step in computing the gradient histogram is simply quantizing
                 22 % the magnitude M into nOrients [hxw] orientation channels according to the
                 23 % gradient orientation. The magnitude at each location is placed into the
                 24 % two nearest orientation bins using linear interpolation if softBin >= 0
                 25 % or simply to the nearest orientation bin if softBin < 0. Next, spatial
                 26 % binning is performed by summing the pixels in each binSize x binSize
                 27 % region of each [hxw] orientation channel. If "softBin" is odd each pixel
                 28 % can contribute to multiple spatial bins (using bilinear interpolation),
                 29 % otherwise each pixel contributes to a single spatial bin. The result of
                 30 % these steps is a floor([h/binSize w/binSize nOrients]) feature map
                 31 % representing the gradient histograms in each image region.
                 32 %
                 33 % Parameter settings of particular interest:
                 34 %  binSize=1: simply quantize the gradient magnitude into nOrients channels

                 36 %  softBin=-1; useHog=2, clip=.2: FHOG features (see fhog.m)
